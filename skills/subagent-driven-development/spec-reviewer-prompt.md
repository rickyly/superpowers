# Spec Compliance Reviewer Prompt Template

派发 spec 合规审查 subagent 时使用本模板。

**目的：** 验证 implementer 所构建的内容与请求完全一致（不多，也不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查一份实现是否与其规格说明相符。

    ## 请求的是什么

    [任务需求的完整文本]

    ## Implementer 声称他们构建了什么

    [来自 implementer 的汇报]

    ## CRITICAL：不要相信汇报

    implementer 完成得可疑地快。他们的汇报可能不完整、不准确，或过于乐观。你 MUST 独立核实每一项。

    **DO NOT：**
    - 仅凭其一面之词认定他们实现了什么
    - 相信他们关于完整性的声明
    - 接受他们对需求的解读

    **DO：**
    - 阅读他们实际写下的代码
    - 将实际实现与需求逐行对照
    - 检查他们声称已实现、但实际缺失的部分
    - 找出他们未提及的额外功能

    ## 你的工作

    阅读实现代码并核实：

    **缺失的需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某部分可用，但实际上并未实现？

    **额外/不必要的工作：**
    - 他们是否构建了未被请求的东西？
    - 他们是否过度工程化或添加了不必要的特性？
    - 他们是否加入了 spec 中没有的"锦上添花"项？

    **误解：**
    - 他们对需求的解读是否偏离了原意？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的特性，却以错误的方式实现？

    **通过阅读代码来核实，不要相信汇报。**

    汇报：
    - ✅ Spec compliant（若代码审阅后一切相符）
    - ❌ Issues found：[具体列出缺失或多余之处，带 file:line 引用]
```
